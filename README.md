# VidMotion — AI 视频广告生成平台

> 一个面向中小广告主的 AI 视频广告生成工具的产品原型。上传商品图 → AI 自动分析 → 输出可投放视频。制作周期从 3-5 人天降至 0.5 人天。

**这是一个 PM 作品集项目**，重点展示产品逻辑、交互设计与 AI 产品思维，AI 生成部分当前使用 Mock 数据。

---

## 项目背景

视频广告的 ECPM 显著高于图片广告，但中小广告主面临视频制作成本高（单条数千元）、周期长（3-5 人天）的资源约束。市面上的 AI 视频生成工具普遍要求用户撰写长篇 prompt，对缺乏专业能力的中小店主极不友好。

VidMotion 的核心产品理念：**AI 产品的价值不是让用户学会用 AI，而是让用户不需要知道 AI 的存在。** 上传图片后，系统自动分析主体、匹配运动效果、生成可投放视频，将 prompt 的复杂度从用户侧转移到产品侧。

---

## 核心功能

| 功能 | 说明 |
|------|------|
| **图片智能分析** | 上传商品图后 AI 自动识别主体、品类、背景类型，推荐最佳运动效果和强度 |
| **可视化效果选择** | 6 种运动效果用视觉卡片展示，替代文字 prompt 输入，中小店主可直接浏览选择 |
| **模板市场** | 按行业和场景分类的预设模板，展示 CTR 提升数据，一键「做同款」 |
| **质检评分体系** | 从内容准确性、动作连贯性、色彩一致性三个维度评分，达标方可投放 |
| **效果看板** | 生成量趋势、各场景 CTR 对比、历史记录，还原数据驱动的产品运营闭环 |

## 产品设计说明

### 核心设计原则

1. **渐进式复杂度**：新用户选模板即可零输入生成，进阶用户可打开高级设置调整 prompt
2. **可视化替代文本化**：运动效果用卡片展示，用户所见即所得，不需要描述"镜头怎么动"
3. **辅助式生成**：AI 主动分析图片并给出建议，用户只需确认或微调

### 为什么不做传统 Prompt 文本框

当前主流 AI 视频工具（Runway、Pika 等）沿用文生图的「一个大文本框」范式。但视频生成需要同时描述主体、运动、镜头、时序、风格，维度远多于图片生成，试错成本也高得多。中小店主是 prompt 能力最弱的群体——这意味着产品设计不应该「教用户写 prompt」，而应该「让用户不需要写 prompt」。

### 质检评分维度

| 维度 | 含义 | 阈值 |
|------|------|------|
| 内容准确性 | 主体识别完整性、特征保留度 | ≥ 85 |
| 动作连贯性 | 帧间过渡流畅度、无闪烁/跳帧 | ≥ 80 |
| 色彩一致性 | 全片色调统一、主体色差可控 | ≥ 85 |
| 综合评分 | 加权平均 | ≥ 80 可投放 |
